파이썬 입문 강의영상 내용

개발환경 구축방법이 많다.(공통 맥락을 파악하면 이해하기 쉽다.)

1세대 프러그래머

python: 윈도우에서 anaconda prompt 실행 ->python ipython: 윈도우에서 anaconda prompt 실행 ->ipython(문법을 알아보기 쉬움-문법강조, ?list와 같이 도움말 기능 동작) 둘 다 저장기능이 없다.

2세대 프로그래머

메모장 + anaconda prompt 이용(.py를 이용한 python활용) 메모장에 코딩을 하고 파일이름을 mysum.py로 저장 anaconda prompt에서 mysum.py파일이 저장된 폴더로 이동 -> 실행

3세대 프로그래머

메모장 + ipython ipython에서 !python mysum.py를 입력하면 anaconda prompt에서 python mysum.py를 입력한 것과 같은 효과 ipython에서 %run mysum을 입력하면 메모장에서 mysum.py에 입력된 내용을 복사해서 ipython에 붙여넣어 실행한것과 같은 효과

4세대 프로그래머

IDE(통합개발환경)를 사용 메모장과 ipython을 하나로 통합한 프로그램이 등장 = jupyter notebook, jupyter lab, idle, VScode 등.. +)주피터의 트릭(실제로 주피터는 ipython에 기생할 뿐 아무런 역할도 안함) *중요한 사실들 -IDE는 내부적으로 연산을 수행하는 능력이 없다. -주피터에서 커널을 재시작한다는 의미는 메모장이 열린채로 ipython을 껏다가 다시 실행한다는 의미 -주피터는 단순히 메모장의 내용을 복사하여 붙여넣는 기계, 하나의 IDE가 여러개의 언어와 연결될 수 있다. -Jupyter lab이라는 프로그램은 크롬에 있는 내용과 ipython간의 통신을 제어하는 프로그램일 뿐이다.

5세대 프로그래머

가상컴퓨터(anaconda), 원격컴퓨터(server), 클라우드컴퓨터(colab)의 개념 등장 모델1: 원격컴퓨터 준비상태: 전북대 컴퓨터(pyton을 실행, 인터넷 연결O), 우리집 컴퓨터(크롬 실행, 인터넷 연결O) 우리집 노트북->전북대 컴퓨터->우리집 노트북 모델2: 원격컴퓨터+가상컴퓨터(환경) 전북대 컴퓨터(논리적으로 3개로 분리 각각 base, py39jl17, py38r40) 논리적으로 분리하면 서로 아무런 타격이 없다. 모델3: 가상컴퓨터 현재 컴퓨터는 base, py39 2개의 컴퓨터로 나누어져 있음 py39에만 주피터랩을 설치 py39, ipython과 여러분의 크롬창이 서로 통신하면서 실습 장점: 서로 다른 환경에 서로다른 파이썬과 R을 설치할 수 있음 = 패키지간의 충돌 최소화 모델4: 클라우드 -사례1 성능이 그저그런 컴퓨터 27개 한사람당 27/5(=5.4)대의 컴퓨터식 할당 -사례2 구글코랩 구글에 여러가지 성능을 가진 컴퓨터 n대 m명의 사람이 n대의 컴퓨터에 접속
